# Paye ta graille — Brand book (v1)

**Objectif** : marque **forte**, **mémorable**, **assumable** — sans dilution en “app générique”.  
**Alignement** : slogans & angles pub → `MARKETING_POSITIONING.md` · voix micro → `UX_COPY_SYSTEM.md` · **identité visuelle complète** → `IDENTITE_VISUELLE_COMPLETE.md` · tokens app → `DESIGN_SYSTEM.md`.

---

## 1) Perception du nom

### Forces
- **Oral** : se dit vite, effet **bouche-à-oreille** naturel (FR).  
- **Concret** : “graille” = **nourriture + désir + quotidien** — pas corporate.  
- **Double lecture** : “paye” = **payer** (cadre repas) + sonorité **“paie”** (oral) → **pacte social** autour de l’assiette.  
- **Mémorisable** : rare en “marque produit” ; déjà une **personnalité**.

### Risques / malentendus
- **Orthographe** : “paye / paie” — gérer **SEO + stores** (alias, sous-titre explicite).  
- **Sens “transaction humaine”** : si mal cadré, peut sonner **froid** — contrer par **copy** (“une table”, “ensemble”).  
- **International** : nom **très francophone** → futur : sous-marque locale ou **claim anglais** séparé (pas besoin jour 1).

### Sous-titre “anti-malentendu” (stores / landing)
Ex. : **« Repas réels entre humains. »** ou **« Ne mange plus seul. »** — ancre le **social IRL**, pas “qui paie quoi” seul.

## 3) Storytelling de marque (arc narratif)

### Chapitre 1 — Le constat (vrai)
Beaucoup veulent **manger avec quelqu’un** sans passer par la **performance** du dating ni l’effort d’**organiser un groupe**.

### Chapitre 2 — L’insight
Le **repas** est le **prétexte le plus naturel** : tout le monde comprend “on mange ensemble”. Ça **réduit la gêne** et **cadre** l’intention.

### Chapitre 3 — La promesse
**Ne mange plus seul.** Pas “trouve l’amour” — **trouve une table**.

### Chapitre 4 — La preuve (terrain)
Des **tables réelles**, des **lieux publics**, des **intentions claires** (J’invite · On partage · Je me fais inviter).

### Chapitre 5 — L’avenir (vision)
Une **habitude locale** : mercredis, partenaires, événements — une **infrastructure du repas social** sans remplacer les amis, mais en **ajoutant** des sièges à la table.

**Anti-histoire** (ce que la marque **refuse**) : promesse magique, honte des utilisateurs, “achat de relation”, nutrition punitive.

## 4) Territoire émotionnel

### Axes (2×2 simplifié)

```
        Chaleur / convivialité
              ↑
              │
   Nostalgie  │  Joie légère
   “comme à   │  “ce soir on sort”
   la maison” │
←─────────────┼─────────────→
   Sérieux    │   Spontanéité
   confiance  │   “pourquoi pas”
              │
              ↓
        Froid / tech froide   ← INTERDIT territoire
```

**Zone sweet spot** : **chaleur** + **légèreté** + **confiance** (sans lourd moralisateur).

### Émotions à activer
- **Soulagement** : “je ne suis pas seul ce soir”.  
- **Curiosité** : “qui sera à la table ?”  
- **Fierté douce** : “j’ai osé”.  
- **Appartenance** : “mon quartier mange ensemble”.

### Émotions à ne pas exploiter
- Honte (“tu es antisocial”).  
- Peur toxique (“tu rates ta chance”).  
- Jalousie / comparaison.

## 5) Archétype de marque (outil interne)

**Mélange dominant** : **L’Hôte** (accueil, cadre) + **Le Complice** (humour léger, pas le prof).  
**Pas** : le Séducteur agressif, le Gourou, le Juge.

---

## 6) Signature sensorielle (rappel)

- **Visuel** : crème, orange chaud, vert validation — voir `DESIGN_SYSTEM.md`.  
- **Verbal** : tutoiement, phrases courtes, 1 blague max par écran si elle sert la **tension** (pas le décor).  
- **Son** (phase 2) : discret, “restaurant calme”, opt-in.

---

## 7) Check-list “marque forte”

- [ ] 1 phrase + 1 sous-titre **identiques** partout (stores, landing, pitch).  
- [ ] **3 intentions** visuellement iconiques (même ratio partout).  
- [ ] **1 rituel** local nommé (ex. “mercredi table”) quand prêt.  
- [ ] **Preuve** : photos/récits **consentis** sur la ville pilote.  
- [ ] **Interdiction** : promesses non tenues (“moins cher” sans preuve).

- Ajouter des features ≠ ajouter des **promesses** de marque.  
- Si le produit s’élargit (événements, groupe), la marque reste : **table d’abord**.
