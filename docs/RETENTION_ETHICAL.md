# Paye ta graille — Rétention forte **sans** dépendance toxique

**Livrable synthèse** (boucles, scènes émotionnelles, social, progression, récompenses, UX, alertes + filtre « vie réelle » + **carte lecture brief neuro/UX**) : **`SYSTEME_ENGAGEMENT_NATUREL.md`**.

**Mission** : les gens reviennent pour **vivre une table**, pas pour **scroller**.  
**Contraintes** : pas de culpabilisation, pas de FOMO artificiel, pas de streaks punissifs, pas de comparaison sociale cachée.

**Principe directeur** : la “récompense” principale est **hors écran** (repas, lien). L’app est un **pont**, pas une destination addictive.

---

## 1) Boucles d’engagement **naturelles** (saines)

### Boucle A — “Prochaine table” (rétention par **engagement réel**)
**Déclencheur** : fin de repas satisfaisant.  
**Action** : proposer **un créneau** ou **une personne** (max 2 options).  
**Récompense** : anticipation sociale **légitime**.  
**Garde-fou** : bouton **“Plus tard”** toujours visible ; pas de compte à rebours mensonger.

### Boucle B — “Rituel local” (habitude calendrier)
Ex. **Table du mercredi** (même quartier).  
**Pourquoi c’est sain** : ancré dans le **temps réel**, pas dans une infinite timeline.

### Boucle C — “Préparation minimale” (responsabilité légère)
Rappels **J-24 / J-2h** uniquement si **repas confirmé** — utiles, pas addictifs.

### Boucle D — “Lien entre deux tables” (V2)
Jauge / compagnon de graille : progression basée sur **IRL complété**, pas sur temps d’écran.

### Boucle E — “Invitation d’un humain réel”
Parrainage **contextuel** après bon moment — social par nature, pas extraction.

---

## 2) Attachement **émotionnel** au produit (sans exploitation)

### Ce qu’on cultive (éthique)
- **Confiance** : “l’app tient ses promesses” (lieu, cadre, respect).  
- **Identité** : “je suis quelqu’un qui ose la table” — **privé**, pas spectacle.  
- **Appartenance locale** : “mon quartier mange ensemble” — **communauté**, pas vanity metrics.

### Ce qu’on évite (toxique)
- Peurs “tu rates ta chance”.  
- Streaks “tu perds tout”.  
- Scores publics de popularité.  
- Notifications nocturnes non critiques.

### Micro-moments émotionnels sains
- Après repas : **gratitude** optionnelle (“Merci pour ce moment”) — pas obligatoire.  
- Après absence longue : **accueil doux** (“Quand tu veux, une table”) — pas reproche.

## 3) Habitudes utilisateur (modèle Fogg adapté)

**B = MAP** (Motivation × Ability × Prompt) — le prompt doit arriver **quand la motivation est naturelle** (faim, solitude, envie de sortir).

| Habitude saine | Prompt acceptable | Prompt toxique |
|----------------|--------------------|-----------------|
| Réserver un créneau hebdo | rappel **si** abonnement créneau | spam quotidien |
| Revoir un contact graille | 1 nudge / semaine max | “X ignore tes messages” |
| Découvrir une table ouverte | lors ouverture app + géoloc | faux badges “urgent” |

**Ability** : réduire les étapes jusqu’à “**1 tap** pour dire oui à une proposition crédible” (concierge au début).

---

## 4) Moments de **retour** dans l’app (où la valeur est claire)

1. **Avant le repas** : confirmation, lieu, hôte — *utilité*.  
2. **Jour J** : accès carte, message court — *réduction stress*.  
3. **Après repas** : feedback 10 secondes + “prochaine table ?” — *fermeture de boucle*.  
4. **Créneau rituel** : mercredis — *calendrier*, pas scroll.  
5. **Événement** (V2) : billet / info pratique — *IRL*.

**Règle** : si l’utilisateur ouvre l’app **sans** repas à venir, l’écran d’accueil doit proposer **une action IRL** (créneau, table ouverte, inviter) — pas un feed infini “pour te retenir”.

---

## 5) Anti-patterns (check-list produit)

- [ ] Pas de streak avec perte  
- [ ] Pas de “en ligne maintenant” sur autrui (stalking)  
- [ ] Pas de notifications marketing nocturnes  
- [ ] Pas de récompenses aléatoires type casino  
- [ ] Pas de chantage à l’invitation  
- [ ] **Réglage global** : fréquence des rappels (déjà prévu spec)

## 6) Métriques de rétention **saines** (à piloter)

- **Repas complétés / utilisateur actif / mois** (North Star comportementale)  
- **Délai moyen jusqu’au 2e repas** (qualité du produit)  
- **Taux de retour après 28 jours** parmi ceux qui ont complété **1** repas  
- **Temps dans l’app** : **non** objectif principal — surveiller plutôt **taux show-up** et **répétition IRL**
