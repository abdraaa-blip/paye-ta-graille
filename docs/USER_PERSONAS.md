# Paye ta graille — Personas utilisateurs (recherche & design)

**Objectif** : concevoir pour des **humains réels**, pas pour “l’utilisateur moyen”.  
**Usage** : parcours UX, copy, règles de sécurité, priorisation features, tests recrutement.

**Note méthodo** : personas = **hypothèses** à valider par entretiens (voir `PROMPT_LIBRARY_EXTENDED` D1). Ajuster chiffres d’âge / ville selon pilote.

---

## Persona 1 — **Sofia, 26, en alternance, coloc**

**Contexte** : vit en colocation ; entourage étudiant mais **souvent dispo le soir en décalé** ; budget serré.  
**Motivations** : sortir sans dépenser comme en “grosse soirée” ; **rencontrer** sans passer pour “en quête”.  
**Peurs** : gêne au 1er contact ; tomber sur quelqu’un **lourd** ; se sentir obligée de “faire date”.  
**Usage app** : intention **On partage** ou **Je me fais inviter** selon semaine ; filtres **budget** ; préfère **créneaux courts** ; lit les **tags Graille** avant d’accepter.  
**Objections** : “C’est encore une app de **drague** déguisée ?” — “Et si la personne **ne paye pas** sa part ?” — “Je veux pouvoir **partir** proprement.”

**Implications produit** : clarifier non-dating ; mode partage **explicite** ; **lieu public** ; sortie / signalement visible.

---

## Persona 2 — **Marc, 38, télétravail, divorcé récent**

**Contexte** : semaines **solo** ; amis en couple avec rythme différent ; veut du **contact humain** sans projet amoureux affiché.  
**Motivations** : “**Ne pas manger seul**” le midi ou le jeudi soir ; conversations **adultes**.  
**Peurs** : jugement (“t’es sur une app…”); conversation **politique tendue** ; silence gênant.  
**Usage app** : intention sociale **Ami / ouvert** ; **J’invite** occasionnellement pour “casser” la dynamique honte ; apprécie **événements** ou **tables du mercredi**.  
**Objections** : “Je ne veux **pas** de photos ridicules sur les réseaux.” — “Confidentialité : qui voit mon **boulot** ?”

**Implications produit** : **discrétion** marketing ; contrôle visibilité ; **prompts de conversation** optionnels (légers).

---

## Persona 3 — **Lina, 22, nouvelle en ville (études / premier job)**

**Contexte** : **0 réseau local** ; fatigue des groupes WhatsApp anonymes.  
**Motivations** : découvrir **restos** + **faire connaissance** vite ; sortir de la **solitude** du studio.  
**Peurs** : sécurité **femme seule** ; arnaques ; se perdre / mal choisir le lieu.  
**Usage app** : beaucoup **scroll** discover au début ; besoin de **rassurance** (lieu public, règles, “première fois ?”); préfère **On partage** pour symétrie.  
**Objections** : “C’est **dangereux** avec un inconnu ?” — “Je ne connais pas les **quartiers**.”

**Implications produit** : **carte** + quartiers sûrs perçus ; **partenaires** ; option “**1er repas** avec créneau encadré / hôte” en pilote.

---

## Persona 4 — **Julien, 45, restaurateur / manager de salle (côté pro)**

**Contexte** : **pas** utilisateur final classique mais **acteur** écosystème.  
**Motivations** : remplir **creux** ; groupes **calmes** ; image **conviviale**.  
**Peurs** : clients **bruyants** ou **no-show** ; mauvaise **presse** si incident.  
**Usage** : portail léger / contact terrain ; **créneaux** ; fiche **allergènes** accessible.  
**Objections** : “Je n’ai pas le temps d’**apprendre une app**.” — “Je ne veux pas de **rabais** permanent.”

**Implications produit** : onboarding **1 page** + **QR** ; pas de friction caisse au début (voir `RESTAURANT_PARTNERSHIPS.md`).

---

## Persona 5 — **Amine, 31, halal / pratique modérée, ingénieur**

**Contexte** : veut **respect du cadre** alimentaire sans devoir **expliquer 20 fois**.  
**Motivations** : rencontres **sans awkwardness** autour du repas ; **transparence** des options halal / sans alcool.  
**Peurs** : malentendu ; être mis dans une case **religieuse** en public ; lieu inadapté.  
**Usage app** : tags **Halal / sans alcool** ; filtre **“table 100% halal”** ou “**respect du cadre**” ; préfère **lieux connus** ou partenaires.  
**Objections** : “Les gens vont **mal comprendre** mes filtres.” — “Je veux éviter le **débat** à table.”

**Implications produit** : tags **neutres et respectueux** ; suggestion **lieux** cohérents ; **signalement** si débordement.

---

## Persona 6 — **Chloé, 29, végétarienne, créative freelance**

**Contexte** : veut **social** et **découverte** ; parfois veut **“convertir”** sans prêcher (plutôt **faire découvrir**).  
**Motivations** : tables **végé** ou **“découverte”** ; événements **street food végé**.  
**Peurs** : blagues lassantes ; resto sans **vraies** options ; pression sociale.  
**Usage app** : section **Graille** riche ; crée des **repas ouverts** “qui est chaud pour…” ; utilise **feed** en V1.5.  
**Objections** : “Je ne veux pas d’un **débat viande** à chaque fois.”

**Implications produit** : intention **“découverte”** vs **“100% végé”** ; modération **topics sensibles** dans feed.

---

## Persona 7 — **Thomas, 34, sportif “healthy”, introverti**

**Contexte** : manger **propre** sans faire **app santé** ; besoin de **structure** pour socialiser.  
**Motivations** : repas **légers** + lien ; évite **alcool** en semaine.  
**Peurs** : jugement sur le **corps** / régime ; soirées trop **alcoolisées**.  
**Usage app** : tags **healthy** ; intention **On partage** ; horaires **tôt** ; évite **“gros gourmand”** en match si mal interprété.  
**Objections** : “Je ne veux pas que ça devienne une **app fitness**.” — “Je n’ai pas envie de **justifier** ce que je mange.”

**Implications produit** : ton **non moralisateur** ; pas de **calories** imposées ; filtre **ambiance** (calme).

---

## Persona 8 — **Élodie, 52, séparée, cadre, quartier aisé**

**Contexte** : réseau stable mais **routines** ; curiosité **discrète** ; exige **sécurité** et **élégance** de l’expérience.  
**Motivations** : **nouvelles têtes** ; sorties **qualité** ; pas de “jeunes qui crient”.  
**Peurs** : **inadéquation** d’âge ou de style ; **volubilité** excessive ; mauvais lieu.  
**Usage app** : filtres **âge / budget / style resto** ; préfère **J’invite** pour **contrôler** le cadre ; lit **avis** / partenaires.  
**Objections** : “C’est pour **quel âge** ?” — “Je ne veux pas me sentir **ridicule**.”

**Implications produit** : **préférences d’âge** (avec équité — éviter exclusion abusive) ; **partenaires** premium ; copy **sobre**.

## Anti-personas (à ne pas optimiser en premier)

1. **Prédateur / harceleur** — à **détecter et exclure** (produit + modération), pas “persona à satisfaire”.  
2. **Gratuit-only / arnaque** — limites invitations, signalement.  
3. **“Influenceur”** qui veut audience sans repas réel — règles anti-spam feed.
