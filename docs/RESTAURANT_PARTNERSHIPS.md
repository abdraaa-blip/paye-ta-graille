# Paye ta graille — Partenariats restaurants (écosystème durable)

**Mission** : intégrer les **professionnels** sans transformer Paye ta graille en “app de résa générique”.  
**Principe** : valeur claire pour le resto = **créneaux remplis** + **image conviviale** + **trafic qualifié** (groupes prévisibles), en échange de **cadre** (accueil, infos pratiques, parfois offre menu).

---

## 1) Pourquoi un restaurant accepterait (arguments terrain)

| Besoin resto | Ce que Paye ta graille apporte |
|--------------|--------------------------------|
| **Lissage des creux** | tables groupées sur **créneaux** ciblés (ex. mardi/soir tôt) |
| **Remplissage sans rabais agressif** | clients **motivés** (social) plutôt que chasseurs de promo |
| **Visibilité locale** | mise en avant auprès d’une **communauté** (ville pilote) |
| **Image** | “tables conviviales”, **contenu** autorisé (photos soirée) |
| **Simplicité opérationnelle** | **menu fixe** ou **formule** possible = service plus fluide |

**Objections fréquentes** (à anticiper) : peur du **désordre**, **no-show**, **avis négatifs** si mauvais match, charge **allergènes** (rappel : information consommateur = responsabilité **exploitant** sous [FIC UE](https://food.ec.europa.eu/food-safety/labelling-and-nutrition/food-information-consumers-legislation_en) — l’app aide à **collecter des besoins**, pas à remplacer l’info légale).

---

## 2) Modèle **gagnant-gagnant** (évolutif)

### Phase pilote (0 frais ou quasi)
- **1 partenaire “pilote”** / quartier : soirée **table Paye ta graille** hebdo.  
- Échange : **salle / créneau** contre **visibilité** + **première preuve** (photos, témoignages).  
- Option : **menu à prix fixe** (réduit friction “qui paie”).

### Phase croissance
- **Package “Soirée tables”** : X couverts, créneau, mise en avant dans l’app + réseaux.  
- **Commission sur billet** si événement curé (V2) — **transparent** pour le client.  
- **Mise en avant sponsorisée** (liste “partenaires” dans la zone) — **pas** masquer le caractère sponsorisé.

### Ce qu’il faut éviter
- Promesse de **déferlentation** sans preuve locale.  
- Modèle uniquement **% addition** complexe à tracer sans intégration caisse (plus tard).

---

## 3) Intégration dans **l’app** (simple V1 → riche V2)

### V1 (léger, réaliste)
- Fiche **Lieu partenaire** : nom, adresse, lien Maps, **créneaux disponibles**, tag “partenaire”.  
- Sélection lieu lors du repas : **liste** (Places + override partenaires en priorité locale).  
- **QR** en caisse : “vous êtes une table Paye ta graille ?” → **info allergènes / menu** (renvoi vers procédure du resto).

### V1.5
- **Créneaux réservables** (slots) côté resto (tableau admin simple ou Google Sheet → import).

### V2
- **Dashboard resto** (portail) : slots, capacité, no-show, demandes groupe.  
- **API** ou intégration résa si volume (TheFork-like, outils internes — hors scope initial).

---

## 4) Avantages concrets pour eux (pitch 30 secondes)

1. **Trafic qualifié** : groupes qui **viennent pour manger ensemble**, pas “passage”.  
2. **Mise en avant** : badge + **story** commune (avec accord).  
3. **Prévisibilité** : créneaux et **effectifs** annoncés.  
4. **Contenu social positif** : “soirée tables” = **image chaleureuse**.  
5. **Feedback direct** (privé) avec l’équipe Paye ta graille pour ajuster le format.

---

## 5) Onboarding resto **simple** (check-list terrain)

### Étape 1 — Qualification (15 min)
- Contact **décideur** (manager / gérant).  
- Questions : capacité **groupes**, créneaux creux, **menu fixe** possible ?, **allergènes** comment gérés.

### Étape 2 — Accord cadre (1 page)
- **Créneaux** proposés + **effectif max** + **durée** + **point d’accueil** (nom du staff référent).  
- **Règles** : lieu public, respect, signalement côté app.

### Étape 3 — Fiche lieu dans l’app
- Photo, horaires partenaires, lien tel pour **allergènes** / **résa** si besoin.

### Étape 4 — Première soirée test
- **10–14 couverts**, hôte Paye ta graille présent, **débrief** 24h après.

### Étape 5 — Itération
- Ajuster créneau / menu / consigne staff.

**Matériel à fournir au resto** : **1 flyer A5** (QR + phrase) + **script staff** 3 lignes (“demandez les allergies au passage, voici la fiche”).

---

## 6) KPI partenariat (pour toi et le resto)

- **Taux de show-up** des tables partenaires  
- **Panier moyen** (si mesurable avec accord) ou **satisfaction** post-soirée  
- **Répétition** : le resto redemande un créneau ?
